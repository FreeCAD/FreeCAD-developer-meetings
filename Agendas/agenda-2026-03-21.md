2. `App::Link` maintenance issues (pieterhijma)
   - `App::Link` has many quirks that make FreeCAD difficult to maintain
   - Quirks:
	 - copy on change (2 different modes)
	   - Enabled/Owned ("normal" copy-on-change)
	   - Tracking (maintain a copy that gets updated on each change)
	   - via subshape binders
	 - subshape binders (special kind of copy-on-change, via temporary hidden files)
	 - scaling
	 - elements
	 - scaling per element
	 - switching between elements (and changing semantics, changing addressing of elements)
	 - link transform for custom placement
	 - custom execute functions
	 - 3 different link groups (besides the elements)
	   - Simple Group
	     - different link modes (none, auto delete, auto link, auto unlink)
	   - Group with transform links
	 - subclassed links
	   - draft arrays (custom element switching)
	   - bim links (custom execute function)
   - Relevant recent PRs:
	 - [#28104](https://github.com/FreeCAD/FreeCAD/pull/28104)
	 - [#26085](https://github.com/FreeCAD/FreeCAD/pull/26085)
   - CQWG discussion on Discord: [Dealing with "quirkful" legacy code](https://discord.com/channels/870877411049357352/1354091480393842799/threads/1479117214853628076)

3. Versioning FreeCAD files (pieterhijma)
   - Looked into versioning of FreeCAD files as part of the [Lens/FreeCAD integration project](https://nlnet.nl/project/Lens-FreeCAD-integration/)
	 - Issues
	   - [#11936](https://github.com/FreeCAD/FreeCAD/issues/11936) - [(my comment)](https://github.com/FreeCAD/FreeCAD/issues/11936#issuecomment-4054297851)
	   - [#9432](https://github.com/FreeCAD/FreeCAD/issues/9432)
	 - Versioning in general, not only for git
	 - Collected lots of information, see [(my comment)](https://github.com/FreeCAD/FreeCAD/issues/11936#issuecomment-4054297851)
   - Already a nice git filters/hooks solution: https://github.com/MikeOpsGit/GitCAD
   - Wanted to see if better support from FreeCAD files is possible, design goals
	 - small changes to FreeCAD code
	 - backward compatibility
	 - forward compatibility
   - Avenues considered:
	 - Realthunder's save as directory (too much code)
	 - Discardable BRep files (my focus)
	 - Zero compression (zip can do that, still some binary, perhaps tar?, out of scope for me)
   - Discardable BRep/binary files: https://github.com/FreeCAD/FreeCAD/pull/28312
	 - User-defined document cache
	 - `Part::Feature` has a property `CanComputeShape` (false by default)
	 - setting the user-defined document cache acts as a switch
	 - BRep files and thumbnails are stored in the cache (if set and the shapes can be recomputed)
	 - document cache directory can be ignored, part of git lfs, shared drive
